Common Questions

  • After completing the analysis in the VRDC, how do you export the data?
    • CCW provides a file transfer system that allows a certain number of export request per seven days on a rolling basis. The number of exports depends on the status of DUA. Researchers and Innovators are allowed to export three and six times per week, respectively The total size of files cannot exceed 1 gigabyte per each export. Before exporting any output cells containing less than 11 observations and any personal identifiers must be redacted. CCW provides a detailed guide about the requirements and the process of downloading output.
  • What types of files can be exported from the VRDC?
    • Most common file types can be exported (e.g., csv (*.csv), Excel (*.xlsx, *.xls, *.xlsb), SAS data sets (*.sas7bdat), or text formats (*.doc, *.docx, *.rtf, *.pdf, *.lst, *.txt) *.stata, and *.tsv. Some files types such as *.dat, *.do, *.dta, *.fts, *.gph, *.htm, *.html, *.jpeg, *.png, *.pptx, *.pqt, *.tab, and *.zip might increase the output review time.
  • Is there specific analytic software or skills you must have to analyze data in the VRDC?
    • VRDC provides different analytical software. The only software to initially access CMS data is SAS. After accessing the data with SAS you save the data and then use Python, R, or Stata to perform empirical analysis. To use Python, R, or Stata you need either an analytic container or data bricks. Our workflow is first to create the analysis data set in SAS and then use Python, R, or Stata depending on the team member.
  • Are there rules and/or guidelines investigators must follow to access the VRDC?
    • Investigators must follow strict rules when accessing the data through VRDC. Researcher must be physically present in the U.S. or U.S. territories, meet CCW security requirements (annual security training and HIPAA compliance), and comply with CCW data output review process.
  • Are there any states or years in the MAX data that do not have claims?
    • Yes, for various reasons some states were unable to submit Medicaid data for 2009 to 2011 and so are not in the MAX files. Specifically, the following files: MAX2009.MAXDATA_ME_OT (reference for Maine claims in 2009), MAX2010.MAXDATA_KS_OT, MAX2010.MAXDATA_KS_RX, MAX2010.MAXDATA_ME_OT, MAX2010.MAXDATA_ME_RX (reference for Maine and Kansas claims in 2010), and MAX2011.MAXDATA_ID_OT (paper reference skipping Idaho in 2011) are not available. Individuals from those states may or may not be missing from the eligibility, inpatient, and long term care files as well.

Tips for troubleshooting (Problems you may encounter)

Here are some common errors we encountered:

  • Unable to insert into temporary index while processing summary functions
    • This error appears in SAS EG when you perform an operation that requires too much memory. We encountered this error when we wanted to count the number of unique beneficiaries by some groups (e.g., age, race, sex, state) in a big file (~500 million rows) using proc sql. As an alternative you can first use proc sort nodupkey on the unique identifier and group variable and then proc freq on the group variable to obtain the count.
  • Unable to see any SAS claims files on the VRDC
    • This happens when there is a SAS error or other issue that causes you to be logged out of the SAS instance or your Cloud SAS profile. If that happens, click the bottom right 'No profile selected' button (it replaces 'Cloud SAS') and type in your credentials again under the profiles.
