VRDC Set-up

Logging onto VMware Horizon Client requires using the same LOWERCASE username and passcode, and then there is an Okta 2-factor authentication step. From there, you can log onto the virtual Windows desktop and click to different applications. The File Transfer Request System (FTRS) is available from the desktop as well as the Analytic Container and Databricks.

Working with the Data (SAS EG)

Log into SAS Enterprise using your username in LOWERCASE. In Servers/sasCCW/Libraries, all the data files are displayed. We can only access the data files that we requested (almost all files) but can see all of them. Loading individual files is pretty quick, but you often have to merge 12 month files to get an entire year, which takes some time.

If you ever get logged out or can’t log in in SAS EG, just click “no profile selected” in the bottom right corner and click Cloud SAS. The login screen will pop up again and you can log in.

You can save files in SAS using the folder SHxxxxxx (replace xxxxxx with your DUA number) which can be found under libraries. Otherwise, created files are stored in the WORK directory which uses shared space across all users instead of individual storage. This means data here won't count to storage limits, but your instance and files will be deleted if you use too much of the shared space (more than 5TB).

Analytic Container (R/Python/Stata)

All three are opened through the Jupyter icon on the desktop. For R and Python, you can just open a kernel or notebook. For Stata, you need to be on the equivalent of the home screen to open a Stata instance. You can’t read giant datasets into R/Python/STATA (currently 24GB memory limit). Consider splitting them up by month and reading separately (also get rid of any columns you don’t need before exporting from SAS).

Saving files in the VRDC/file transfers

You can’t copy code out of the VRDC. So, you should keep code in the shared VRDC folder that everyone in your DUA can access. If you want to send it outside, or especially to keep a backup, you need to type it outside of the VRDC and then copy it inside.
We find the easiest and most efficient method is to type everything locally (in any word processor), and then copy all of the script as text. Stata in the VRDC can’t be copied into directly, but you can open Notepad++ (inside the VRDC) and copy into that, and then copy into Stata as a 2-step process.

VRDC additional considerations

Our scripts/data are saved under server “sasCCW/Files/dua_xxxxxx”, which is a shared drive we can all access that has lots of subfolders. This can’t be accessed through Windows file editor, but you can save from SAS or any of the other programming languages, which have that as their home directory.

Everyone also has their own unique filepath to access data on the shared drive. This means that the line below points to a specific filepath on one account, /sas/vrdc/users/zzzzzz/files/dua_xxxxxx, where zzzzzz is the individual username and xxxxxx is the DUA number.
