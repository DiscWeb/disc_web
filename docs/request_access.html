Initial DUA Application

Overview of CMS Data Request Process

  1. When submitting a new request for Research Identifiable (RIF) data, access the required documents on the Request Forms Generator Tool
  2. Complete the requested forms depending on type of data being requested:
    • CMS DUA
    • RIF Application – Attachment A of the CMS DUA
    • Key Personnel Supplement
    • Research Identifiable Files (RIF) Specifications sheet - To identify the cost of obtaining files, complete the specifications sheet

Before Completing the CMS Data Request:

  1. Review and know all the data files needed to complete analysis on the Data File Search webpage
  2. Define and estimate the study cohort by using the following links
    • Crosswalk files for Medicare Data Finder
    • Estimate cohort size
  3. Determine whether to access data via VRDC or physical files.
  4. Estimate cost of obtaining CCW data and/or access to VRDC and secure funding
    • Data Pricing
    • CMS Fee for VRDC access
  5. Review privacy and security compliance requirements.
    • This varies between physical data versus VRDC, but eventually all researchers will have to shift to the VRDC.
  6. Obtain institutional IRB approval.

While Completing the CMS Data Request

  1. Start with the RIF Application (Attachment A):
    • Study Parameters
    • Executive summary
      • Provide a thorough study overview including study objectives, aims, all CMS data files used in the study and how they will be analyzed separately or together.
      • Write a justification for each CMS data file you are requesting including if you need the entire population file or a percentage of the file and why.
      • For purpose of use, provide a statement on how the indicated CMS file will be used for analysis and what information this provides for the analysis.
      • If your study includes external collaborators, individual or organizations, be sure to include a discussion of which organizations are involved in the project, how they are involved, and whether they will access the raw data/data not meeting the cell suppression policy.
        • If the collaborators are accessing the data, CMS will look for a discussion of how they will access the data in the VRDC to ensure adequate data security and compliance are implemented.
        • Also, you will need to complete this form, Collaborating Organization Supplement for the application
    • Dissemination and Reporting Findings
    • Project Staff
    • Data Management Plan
  2. Complete the RIF Data Use Agreement
  3. Fill out the RIF Specifications Worksheet, ensuring consistency with the RIF Application.
    • Make sure that you start each tab with row 1
    • Double check that the information in the RIF Specification Worksheet is the same as what is in the RIF Application as CMS uses to generate an invoice and to collect data extraction information.
    • Our team has occasionally not gotten files because they were included in the DUA file but weren't checked on the specificaiton worksheet, so we had to do an amendment.
  4. Prepare the Key Personnel Statement.
    • Review the Role Definitions list from RESDAC to ensure you are accurately listing the key personnel involved in the project.

When Submitting the CMS Data Request

Follow the instructions on this page and submit all required documents to resdac@umn.edu

Tips for Completing a Successful Request

  • Ensure you allocate funds for VRDC access if applying for a grant.
  • Follow the latest version of all forms and checklists.
  • DUA and VRDC access must be renewed annually for each user.
  • Maintain consistency across all submitted documents.
