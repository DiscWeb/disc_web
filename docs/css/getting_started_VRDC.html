Getting Started in VRDC

Use the CCW guides online to set up the VRDC. CCW has instructions about file transfers for uploading/downloading which are important to refer back to.

SAS Enterprise Guide (EG):

  1. Log into SAS Enterprise using your username in lowercase.
  2. In Servers/sasCCW/Libraries, all the data files are displayed.
  3. We can only access the data files that we requested (almost all files) but can see all of them.
  4. You can’t move things into different folders because it will copy everything to memory.
  5. If you ever get logged out or can’t log in in SAS EG, just click “No Profile Selected” in the bottom right corner and click Cloud SAS. The login screen will pop up again, and you can log in.
  6. Saving files in SAS: Save to the folder SH058994 which can be found under libraries.

R/Python/Stata Usage:

  • All three are opened through the Jupyter icon.
  • For R and Python, you can just open a kernel or notebook. For Stata, you need to be on the equivalent of the home screen to open a Stata instance.
  • You can’t read giant datasets into R/Python (24GB memory limit). Consider splitting them up by month and reading separately. Also, remove any columns you don’t need before exporting from SAS.

Other Coding Considerations:

You can’t copy code out of the VRDC. So, keep it in a shared folder in the VRDC that everyone can access. If you want to send it outside, or especially to keep a backup, you need to type it outside of the VRDC and then copy it inside.

The easiest and most efficient method is to type everything locally (in any word processor), and then copying all of the script as text. In Stata, the VRDC can’t be copied into directly, but you can open Notepad++ (inside the VRDC) and copy into that, then copy into Stata as a two-step process.

File Locations:

Our scripts/data are saved under server sasCCW/Files/dua_058994, which is a shared drive we can all access that has lots of subfolders. This can’t be accessed through Windows file editor, but you can save from SAS or any of the other programming languages, which have that as their home directory.

Everyone also has their own unique filepath to access data on the shared drive. This means that the line below points to a specific file on one account, but you would need to replace xxxxxx with your 6-digit username.

/sas/vrdc/users/xxxxxx/files/dua_058994/data/oud_definition/data/output/MA_final_OUD_&nextyr..csv
